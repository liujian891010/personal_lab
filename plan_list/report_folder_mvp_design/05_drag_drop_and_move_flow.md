# Phase 5: 拖拽移动与归档交互

## 目标

增加“拖动报告到文件夹”的直接交互，降低归档成本，同时保留非拖拽的显式移动入口。

## 前置条件

- 文件夹导航 UI 已存在
- 报告移动 API 已可用

## 任务清单

- 为报告卡片增加拖拽源能力
- 为文件夹导航项增加放置目标能力
- 拖拽过程增加视觉反馈：
  - 当前拖拽态
  - 可放置态
  - 放置成功反馈
- 保留备用移动入口：
  - 卡片菜单中的“移动到文件夹”
  - 详情页中的“移动到文件夹”
- 明确拖拽后的刷新策略：
  - 本地状态直接更新
  - 必要时回源刷新
- 支持移动到：
  - 某个自定义文件夹
  - `Unfiled`

## 交付物

- 拖拽交互实现
- 报告手动移动入口
- 成功/失败反馈机制

## 验收标准

- 用户可将报告卡片拖放到文件夹中完成归档
- 拖拽成功后，当前列表和文件夹计数同步更新
- 拖拽失败时有明确提示，且不会造成前端状态错乱
- 不使用拖拽时，仍可通过普通操作完成移动

## 风险与注意事项

- HTML5 Drag and Drop 在不同浏览器体验不一致，MVP 需优先保证桌面端
- 若列表支持分页，拖拽后当前页数据和总数要同时修正
- 触屏端拖拽可以暂不做，避免把 MVP 范围拉大
