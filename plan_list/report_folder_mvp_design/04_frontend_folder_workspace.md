# Phase 4: Reports 页面文件夹工作区

## 目标

在现有 `reports` 页面加入文件夹导航和管理入口，让用户可以直接浏览文件夹、切换视图、创建文件夹并查看归档结果。

## 前置条件

- 文件夹列表 API 与报告筛选 API 已可用
- 现有前端路由、报告列表和详情页结构稳定

## 任务清单

- 在 `reports` 页面新增文件夹侧栏或顶部分组导航
- 展示项目：
  - `All Reports`
  - `Unfiled`
  - 自定义文件夹列表
  - 每个文件夹的报告数量
- 新增文件夹创建入口：
  - 输入文件夹名称
  - 可选描述
- 新增文件夹管理入口：
  - 重命名
  - 删除
- 报告列表卡片增加文件夹展示
- 报告筛选状态与 hash 路由联动：
  - 例如 `folder_id`
  - 例如 `unfiled=true`
- 上传页面新增文件夹选择控件：
  - 单选下拉
  - 默认空值表示未归档

## 交付物

- `reports` 页面文件夹导航 UI
- 文件夹创建与管理交互
- 上传表单文件夹选择 UI
- 路由与筛选联动

## 验收标准

- 用户可以在前端创建文件夹并立即看到新分组
- 切换文件夹时，报告列表会按预期变化
- 上传表单可选择文件夹
- 报告卡片和详情页能看到所属文件夹

## 风险与注意事项

- 文件夹导航不能压坏现有移动端布局
- 上传页和报告页应复用同一套文件夹数据结构，避免字段名不一致
- 若文件夹过多，后续需要补滚动容器或搜索
