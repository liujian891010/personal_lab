# 05 清理任务、审计与验收

## 目标

补齐后台清理任务、操作审计和验收标准。

## 后台任务

建议新增周期任务：

- 扫描到期待清理报告
- 执行对象删除和本地副本删除
- 更新清理状态
- 记录失败原因，支持重试

## 审计日志

至少记录：

- `userId`
- `report_id`
- 操作类型：`delete_report`
- 删除时间
- `purge_after`
- 是否完成物理清理
- 失败原因

## 验收标准

1. 用户可以从产品界面删除报告
2. 已删除报告不会出现在列表页
3. 已删除报告不会命中搜索
4. 已删除报告不会出现在问答报告召回结果中
5. 保留期内对象仍存在，但前台不可见
6. 超过 `7 天` 后对象和本地副本可被清理
7. 清理失败时可重试
8. 删除审计可追踪到 `userId`

## 建议实施顺序

1. 先做 DB 字段和 API
2. 再做前端删除入口
3. 再做搜索与查询过滤
4. 最后补后台清理任务和审计
