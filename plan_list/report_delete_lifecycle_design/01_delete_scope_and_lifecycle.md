# 01 删除范围与生命周期

## 目标

定义 `report` 删除的业务语义，明确什么叫“删除成功”，以及删除后的生命周期状态。

## 建议语义

删除 `report` 不等于立刻物理擦除，而是：

1. 用户发起删除
2. 系统将报告标记为“已删除”
3. 前台默认不再展示
4. 搜索默认不再命中
5. 进入延迟清理队列
6. 到达保留期后，物理清理对象与文件副本

## 建议状态机

新增报告生命周期状态：

- `published`
- `deleted`

或增加独立删除字段：

- `deleted_at`
- `deleted_by`
- `purge_after`
- `storage_cleanup_status`

推荐方案：

- 保留 `status`
- 额外增加：
  - `deleted_at`
  - `deleted_by`
  - `purge_after`
  - `storage_cleanup_status`

原因：

- `status` 继续表示内容状态
- 删除语义由独立字段承载，更清晰
- 后续恢复、清理、审计更容易扩展

## 默认规则

1. 删除后立即从列表和搜索结果中隐藏
2. 删除后详情页默认返回 `404` 或 `410`
3. 保留期内不物理删除对象
4. 保留期默认为 `7 天`
5. 保留期结束后异步清理对象存储与本地兼容副本

## 与用户隔离的关系

删除必须严格限定在当前 `userId` 工作区下：

- 只能删除当前用户工作区内的报告
- 不能跨用户删除
- 审计日志必须记录 `userId`
