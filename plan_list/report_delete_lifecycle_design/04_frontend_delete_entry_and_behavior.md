# 04 前端删除入口与交互行为

## 目标

在报告列表页和报告详情页提供正式删除入口，并明确用户可见行为。

## 入口建议

1. 报告详情页增加“删除报告”按钮
2. 报告列表卡片增加操作菜单中的“删除”
3. 独立只读详情页默认不提供删除入口

## 交互建议

点击删除后：

1. 弹出确认框
2. 明确提示：
   - 删除后将立即从列表中隐藏
   - 文件将在 `7 天` 后物理清理
3. 用户确认后调用：

```http
DELETE /api/reports/{report_id}
```

## 删除成功后的表现

1. 从当前列表中移除该报告
2. 详情页返回上一页或显示“该报告已删除”
3. 给出 toast 提示

## 第一阶段不做的内容

以下能力可后续补：

- 回收站页面
- 恢复按钮
- 批量删除
- 管理员跨用户删除
