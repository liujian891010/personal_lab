# Phase 6: 配额、保留期与成本控制

## 目标

避免对象存储和临时磁盘随着用户增长无限膨胀，把容量控制和成本控制纳入正式设计。

## 改造步骤

- 设计工作区配额
  - 总容量
  - 单文件大小
  - 单日上传次数或总量
- 设计生命周期策略
  - 原始上传保留期
  - 临时中间文件保留期
  - 删除后冷存档或彻底删除策略
- 去重策略
  - 以 `content_hash` 识别重复文件
  - 元数据引用计数
- 清理任务
  - 定期清理本地 working/tmp
  - 定期清理对象存储过期对象
  - 清理数据库孤儿记录
- 成本监控
  - 按 workspace 汇总容量
  - 按对象类型汇总容量
  - 预警异常增长

## 输出

- 配额字段设计
- 定时清理任务清单
- 存储成本监控指标
