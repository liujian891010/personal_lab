# Phase 3: 上传链路改造到对象存储

## 目标

让上传原文件和抽取产物不再长期保存在应用机磁盘，而是通过统一存储服务写入对象存储。

## 改造步骤

- 上传入口改成两段式
  - 文件先落本地临时目录
  - 校验通过后上传到对象存储
- 写入完成后仅在数据库中保留对象指针
- 抽取文本输出改造
  - 先生成到本地 working 目录
  - 成功后上传到对象存储 `uploads/extracted/`
  - 数据库 artifact 改存对象 key
- 失败补偿
  - 数据库写成功但对象上传失败时回滚
  - 对象上传成功但数据库失败时做孤儿对象清理
- 大文件和断点能力
  - 评估分片上传
  - 统一超时、重试、幂等键
- 权限策略
  - 应用服务端写对象存储
  - 不直接把长期凭证暴露给前端

## 验收点

- 上传完成后本地磁盘只留下临时缓存
- 数据库中能准确定位对象存储文件
- 重试不会生成重复对象
