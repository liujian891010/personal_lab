# Phase 5: 存量迁移、回填与双读切换

## 目标

把现有本地磁盘里的用户数据平滑迁移到对象存储，并且保证迁移期间业务可继续运行。

## 改造步骤

- 盘点现有存量目录
  - `data/workspaces/*/uploads`
  - `data/workspaces/*/raw`
  - `data/workspaces/*/reports`
  - `data/workspaces/*/knowledge`
- 编写迁移任务
  - 扫描文件
  - 上传对象存储
  - 回填数据库对象指针
  - 标记校验状态
- 灰度双读
  - 先写新、读新优先旧兜底
  - 完成批次校验后逐步关闭旧读路径
- 完整性校验
  - 文件数校验
  - `content_hash` 校验
  - 字节数校验
  - 关键业务抽样回放
- 切换计划
  - 新用户先全量走新存储
  - 老用户按 workspace 批次切换
  - 保留回滚窗口

## 回滚原则

- 新对象不删
- 旧本地文件在观察期内保留
- 只回滚读流量，不直接回滚已迁移元数据
