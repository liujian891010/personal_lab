# Phase 7: 运维、审计与上线手册
## 目标

补齐 APPKEY 登录与工作区隔离上线后的可观测性、审计、排障与应急操作能力，确保系统可维护、可定位、可回退。

## 前置条件

- 登录链路、工作区隔离与迁移方案已明确。
- 核心接口与后台任务已按工作区运行。

## 任务清单

- 统一日志字段：
  - `request_id`
  - `user_id`
  - `workspace_id`
  - `route`
  - `status_code`
- 设计关键操作审计：
  - 登录
  - 登出
  - 上传
  - sync
  - compile
  - writeback
  - move folder
  - retry upload
- 健康检查与管理视图增加工作区维度观测信息。
- 设计隔离自检清单：
  - 跨工作区读取验证
  - 后台任务作用域验证
  - 索引与统计隔离验证
- 设计 APPKEY 异常处理策略：
  - 失效
  - 禁用
  - 强制登出
- 编写上线 runbook：
  - 环境准备
  - 配置检查
  - 登录联调
  - 迁移执行
  - 验收核对
- 编写故障排查手册：
  - 登录失败
  - 会话异常
  - 串数据投诉
  - 索引异常

## 交付物

- 上线检查清单。
- 审计字段规范。
- 异常处理手册。
- 运维 runbook 与排障手册。

## 验收标准

- 任一关键请求都能追踪到 `user_id` 与 `workspace_id`。
- 发生串数据投诉时，能快速定位是接口层、Service 层、文件层还是索引层问题。
- 可快速禁用异常 APPKEY 并使会话失效。
- 生产切换前具备完整的上线与回滚操作手册。

## 风险与注意事项

- 若没有统一日志上下文，即使实现了隔离，后续排障也会非常困难。
- 审计日志中不得明文打印 `APPKEY`。
