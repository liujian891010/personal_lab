# Phase 5: Upload 前端工作台

## 目标

为 Upload Center 提供可用的前端入口，支持上传、查看任务状态、阅读抽取结果与跳转关联报告。

## 前置条件

- 上传 API、处理 API、详情 API、报告预览 API 已可用。
- 现有前端路由和样式体系已稳定。

## 任务清单

- 新增页面：
  - `/uploads`
  - `/uploads/:uploadId`
- 列表页展示：
  - 文件名
  - 标题
  - 文件类型
  - 文件大小
  - `upload_status`
  - `processing_stage`
  - `report_id_ref`
  - 更新时间
- 详情页展示：
  - 原始文件信息
  - 状态和阶段
  - 错误信息
  - 原文预览
  - 报告预览
  - 关联报告入口
  - 编译模式和自动编译状态
- 接入操作按钮：
  - Upload
  - Process
  - Retry
  - Open Report
  - View Raw
- 增加基础筛选与搜索。

## 交付物

- Upload 列表页
- Upload 详情页
- 上传入口 UI
- 前后端联调完成的上传工作台

## 验收标准

- 用户可以在前端上传文件并看到新任务出现在列表中。
- 列表页可以按状态查看任务。
- 详情页可以看到原文预览、报告预览和错误信息。
- 任务完成后可从前端跳转到关联报告详情。
- 处理中的任务状态刷新后能正确反映阶段变化。

## 风险与注意事项

- 首版以前端可用为目标，不急于做复杂拖拽、多文件并发体验。
- 页面字段展示要和 API 字段保持一致，不自行造别名。
